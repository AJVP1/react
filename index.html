React

Introducción

React es una librería de JavaScript creada por Meta (Facebook) para construir interfaces de usuario interactivas y dinámicas. A diferencia de un framework completo, React se centra específicamente en la capa de vista, dándote libertad para elegir otras herramientas según tus necesidades.

React se basa en componentes reutilizables que encapsulan la lógica y la interfaz. Utiliza JSX, una sintaxis que permite escribir HTML dentro de JavaScript, y maneja el estado de forma reactiva: cuando los datos cambian, la interfaz se actualiza automáticamente. Esto es posible gracias al Virtual DOM, que optimiza las actualizaciones del DOM real.

Comparado con JavaScript vanilla, React ofrece un código más declarativo (describes qué quieres mostrar, no cómo hacerlo), elimina la necesidad de manipular el DOM directamente con querySelector o innerHTML, y facilita enormemente el mantenimiento de aplicaciones grandes. El ecosistema de React incluye herramientas como React Router, Redux, y Next.js que complementan la librería base.

React es ideal para crear aplicaciones de una sola página (SPAs) donde la interfaz se actualiza dinámicamente sin recargar la página completa. Se utiliza ampliamente en dashboards, redes sociales, plataformas de e-commerce, aplicaciones empresariales, y cualquier proyecto que requiera interfaces complejas e interactivas.

Empezar un proyecto

Para empezar a desarrollar el frontend con React, ubicarse en el root del proyecto y ejecutar: 

npm create vite@latest

Esto creará la carpeta del frontend con configuraciones típicas (node_modules, public, src, etc.). Para producción: 

npm run dev

Dentro de src se crea assets para imágenes y archivos como main.jsx e index.css (principales), y app.jsx/app.css (secundarios).

Ejemplo

function App() {
  const items = ["react", "javascript", "git"];

  return (
    <section>
      <h1>Hola mundo</h1>
      <ul>
        {items.map((item, index) => (
          <li key={index}>{item}</li>
        ))}
      </ul>
    </section>
  );
}

Ventajas

  • Código más declarativo y legible
  • No es necesario manipular el DOM directamente
  • Menos líneas de código para lograr el mismo resultado
  • Mayor facilidad para mantener y actualizar

Componentes

Un componente en React es una función que devuelve lo que queremos que se muestre en pantalla. Suelen escribirse como arrow functions y típicamente empiezan con mayúscula. 

const MyComponent = () => {
  return "I'm a component";
};

Los componentes también pueden devolver “HTML” vía JSX:

const Bienvenida = () => {
  return <h1>Hola, mundo</h1>;
};

JSX parece HTML, pero es una sintaxis que React usa para escribir estructuras tipo HTML dentro de JavaScript.

Props

Las props (“properties”) permiten pasar datos de un componente padre a uno hijo, como parámetros para personalizar y reutilizar componentes. 

const Card = ({ title, content }) => {
  return (
    <div>
      <h2>{title}</h2>
      <p>{content}</p>
    </div>
  );
};

export default Card;

Uso del componente:

import Card from "./components/Card";

function App() {
  return (
    <section>
      <Card title="Card uno" content="Esta es una descripción" />
    </section>
  );
}

export default App;

Estados (useState)

El estado en React almacena información que puede cambiar durante el ciclo de vida de un componente.

Tipos principales

  • Estado local: accesible solo dentro del componente donde se define.
  • Estado global: compartido entre múltiples componentes.

Hook useState

Devuelve:

  • Una variable con el valor actual del estado
  • Una función para actualizar ese valor

Sintaxis básica:

const [valor, setValor] = useState(valorInicial);

Ejemplo: contador

const Counter = () => {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>El contador está en: {count}</h2>
      <button onClick={() => setCount(count + 1)}>Incrementar</button>
      <button onClick={() => setCount(count - 1)}>Decrementar</button>
    </div>
  );
};

Casos de uso comunes

  • Formularios
  • Toggles (activado/desactivado)
  • Filtros
  • Paginación
  • Cargas de datos (loading/success/error)

Ejemplo: toggle

const ToggleButton = () => {
  const [isActive, setIsActive] = useState(false);

  return (
    <button onClick={() => setIsActive(!isActive)}>
      {isActive ? "Activo" : "Inactivo"}
    </button>
  );
};

Ejemplo: formulario

const NameForm = () => {
  const [name, setName] = useState("");

  return (
    <div>
      <input
        type="text"
        placeholder="Ingresa tu nombre"
        onChange={(event) => setName(event.target.value)}
      />
      <p>Hola, {name ? name : "visitante"}</p>
    </div>
  );
};

Efectos secundarios (useEffect)

Los efectos secundarios son operaciones fuera del flujo principal de renderizado (APIs, suscripciones, manipulación del DOM, temporizadores, localStorage, etc.).

Sintaxis

useEffect(() => {
  // efecto
}, [dependencias]);

Si el array de dependencias está vacío ([]), se ejecuta una vez tras el primer renderizado. Si contiene variables, se ejecuta cuando alguna cambie.

Ejemplo

const CounterWithEffect = () => {
  const [count, setCount] = useState(0);

  useEffect(() => {
    console.log(`El contador cambió a ${count}`);
  }, [count]);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Incrementar</button>
    </div>
  );
};

Escenarios comunes

  • Obtención de datos (fetch a APIs)
  • Suscripciones (y limpieza al desmontar)
  • Manipulación del DOM (p. ej., título de la página)
  • Animaciones
  • Persistencia (localStorage)

Estados complejos (useReducer)

useReducer permite definir un reducer que describe cómo cambia el estado ante acciones, similar a Redux, pero integrado en React.

Sintaxis

const [state, dispatch] = useReducer(reducer, initialState);

Ejemplo

const reducer = (state, action) => {
  switch (action.type) {
    case "increment":
      return { count: state.count + 1 };
    case "decrement":
      return { count: state.count - 1 };
    default:
      return state;
  }
};

const [state, dispatch] = useReducer(reducer, { count: 0 });

return (
  <div>
    Contador: {state.count}
    <button onClick={() => dispatch({ type: "increment" })}>
      Increment
    </button>
    <button onClick={() => dispatch({ type: "decrement" })}>
      Decrement
    </button>
  </div>
);

Fetch

Ejemplo de llamado a una API y renderizado del array:

import { useState, useEffect } from "react";

const UserList = () => {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    fetch("https://jsonplaceholder.typicode.com/users")
      .then((response) => response.json())
      .then((data) => setUsers(data))
      .catch((error) => console.error("Error fetching data:", error));
  }, []);

  return (
    <div>
      <h1>Usuarios</h1>
      <ul>
        {users.map((user) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </div>
  );
};

export default UserList;

Manejo de errores y carga

Agregar estados de carga y error:

import { useState, useEffect } from "react";

const UserList = () => {
  const [users, setUsers] = useState([]);
  const [isLoading, setIsLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchUsers = async () => {
      try {
        const response = await fetch("https://api.example.com/users");
        if (!response.ok) throw new Error("Error al obtener los datos");

        const data = await response.json();
        setUsers(data);
      } catch (error) {
        setError(error);
      } finally {
        setIsLoading(false);
      }
    };

    fetchUsers();
  }, []);

  if (isLoading) return <p>Cargando...</p>;
  if (error) return <p>Error: {String(error)}</p>;

  return (
    <div>
      <h1>Usuarios</h1>
      <ul>
        {users.map((user) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </div>
  );
};

export default UserList;

React Router

React Router organiza la navegación entre vistas cuando la aplicación crece, mejorando estructura y mantenibilidad.

Estructura de un React Router

  • Provider: similar a Context; evita prop drilling.
  • Link: reemplaza a <a> para navegar sin recargar.
  • Route: define rutas con path y el componente a renderizar.

Suele incluirse un Menú o Footer que se renderiza en todas las vistas.

Tipos de routers

  • Browser Router: URLs “normales” tipo /home, /blog.
  • Hash Router: usa # en la URL, ej. example.com/#/ruta/blog.
  • Memory Router: mantiene rutas en memoria; útil cuando no se depende de URLs.

Instalación

npm install react-router-dom@latest

Rutas estáticas (HashRouter)

function App() {
  return (
    <>
      <HashRouter>
        <Menu />

        <Routes>
          <Route path="/" element={<HomePage />} />
          <Route path="/profile" element={<ProfilePage />} />
          <Route path="/blog" element={<BlogPage />} />
          <Route
            path="*"
            element={<h1 className="text-3xl font-bold">Not Found</h1>}
          />
        </Routes>
      </HashRouter>
    </>
  );
}

Navegación (NavLink)

const routes = [
  { to: "/", text: "Home" },
  { to: "/blog", text: "Blog" },
  { to: "/profile", text: "Profile" },
];

<ul>
  {routes.map((route) => (
    <li key={route.to}>
      <NavLink
        to={route.to}
        style={({ isActive }) => ({ color: isActive ? "red" : "blue" })}
      >
        {route.text}
      </NavLink>
    </li>
  ))}
</ul>

Rutas dinámicas (useParams)

El slug es parte de la URL que identifica contenido (p. ej., /blog/que-es-react).

Datos

const blogData = [
  {
    title: "¿Qué es React?",
    slug: "que-es-react",
    content: "React es el mejor framework de JavaScript.",
    author: "Juan de Sen",
  },
  {
    title: "¿Qué es Vue?",
    slug: "que-es-vue",
    content: "Vue es un framework progresivo.",
    author: "Otro autor",
  },
];

Lista de posts

const BlogPage = () => {
  return (
    <ul>
      {blogData.map((post) => (
        <li key={post.slug}>
          <Link to={`/blog/${post.slug}`}>{post.title}</Link>
        </li>
      ))}
    </ul>
  );
};

Rutas

function App() {
  return (
    <Router>
      <Routes>
        <Route path="/blog" element={<BlogPage />} />
        <Route path="/blog/:slug" element={<BlogPost />} />
      </Routes>
    </Router>
  );
}

Detalle (useParams)

const BlogPost = () => {
  const { slug } = useParams();
  const post = blogData.find((post) => post.slug === slug);

  return (
    <div>
      <h2>{post.title}</h2>
      <p>{post.content}</p>
      <p>{post.author}</p>
    </div>
  );
};

Autenticación

Página de login

import { useState } from "react";
import { useAuth } from "../auth/auth.jsx";

export default function LoginPage() {
  const auth = useAuth();
  const [username, setUsername] = useState("");

  function login(event) {
    event.preventDefault();
    auth.login({ username });
  }

  return (
    <>
      <h1 className="text-3xl font-bold">Login</h1>

      <form onSubmit={login} className="flex flex-col gap-1.5">
        <label htmlFor="username">Escribe tu nombre de usuario</label>
        <input
          type="text"
          id="username"
          value={username}
          onChange={(e) => setUsername(e.target.value)}
          className="bg-gray-200 rounded-md"
        />
        <button type="submit">Enviar</button>
      </form>
    </>
  );
}

Rutas

<Route path="/login" element={<LoginPage />} />
<Route path="/logout" element={<LogoutPage />} />
<Route path="/profile" element={<ProfilePage />} />

AuthProvider (Context)

import React from "react";
import { useNavigate } from "react-router-dom";

const AuthContext = React.createContext();

function AuthProvider({ children }) {
  const navigate = useNavigate();
  const [user, setUser] = React.useState(null);

  const login = ({ username }) => {
    setUser({ username });
    navigate("/profile");
  };

  const logout = () => {
    setUser(null);
    navigate("/");
  };

  const auth = { user, login, logout };
  return <AuthContext.Provider value={auth}>{children}</AuthContext.Provider>;
}

function useAuth() {
  return React.useContext(AuthContext);
}

export { useAuth, AuthProvider };

Proteger rutas con autenticación

Agregar propiedad private:

const routes = [
  { to: "/", text: "Home", private: false },
  { to: "/profile", text: "Profile", private: true },
  { to: "/blog", text: "Blog", private: false },
  { to: "/login", text: "Login", private: false },
  { to: "/logout", text: "Logout", private: true },
];

Ocultar rutas en el menú según autenticación:

export default function Menu() {
  const auth = useAuth();

  return (
    <nav>
      <ul className="flex flex-row gap-4">
        {routes.map((route) => {
          if (route.private && !auth.user) return null;
          if (route.to === "/login" && auth.user) return null;

          return (
            <li key={route.text}>
              <NavLink
                style={({ isActive }) => ({ color: isActive ? "red" : "blue" })}
                to={route.to}
              >
                {route.text}
              </NavLink>
            </li>
          );
        })}
      </ul>
    </nav>
  );
}

Protección de rutas con redireccionamiento

Para evitar acceso por URL directa, usar un componente de ruta protegida con Navigate. 

const AuthRoute = ({ children }) => {
  const auth = useAuth();

  if (!auth.user) {
    return <Navigate to="/login" />;
  }

  return children;
};

Proteger rutas:

<Route
  path="/logout"
  element={
    <AuthRoute>
      <LogoutPage />
    </AuthRoute>
  }
/>

<Route
  path="/profile"
  element={
    <AuthRoute>
      <ProfilePage />
    </AuthRoute>
  }
/>

Evitar volver al login si ya hay sesión:

if (auth.user) {
  return <p>Ya has iniciado sesion como {auth.user.username}</p>;
}

Roles y autorización

Lista de admins:

export const adminList = ["admin1", "admin2", "admin3"];

Marcar admin al hacer login:

const login = ({ username }) => {
  const isAdmin = adminList.includes(username);
  setUser({ username, isAdmin });
  navigate("/profile");
};

Ejemplo de permiso (solo admins):

export default function BlogPost() {
  const auth = useAuth();
  const { slug } = useParams();
  const post = blogData.find((post) => post.slug === slug);

  return (
    <div>
      <h2>{post.title}</h2>
      <p>{post.content}</p>
      <p>{post.author}</p>

      {auth.user?.isAdmin && <button>Eliminar Post</button>}
    </div>
  );
}
